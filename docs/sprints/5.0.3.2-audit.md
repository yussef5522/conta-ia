## Switcher A (topo) — manter + corrigir

**Arquivos:**
- UI: `components/layout/workspace-switcher.tsx`
- Lógica: `lib/contexts/empresa-context.tsx` (`setCurrentEmpresa`)

**Comportamento atual ao trocar empresa:**
1. `setCurrentEmpresaIdState(id)` — atualiza state interno do contexto
2. Salva localStorage `caixaos:empresa-context:current`
3. POST `/api/empresas/atual` (cookie httpOnly server-side)
4. **NÃO faz `router.refresh()` nem navega.**

**Causa do bug "não recarrega":**

A `/contas-a-pagar/page.tsx` LÊ `searchParams.get('empresaId')` no init e
sincroniza URL via `router.replace(?empresaId=X)` só quando `filters` muda.
Quando Switcher A troca o contexto:
- `currentEmpresaId` no contexto muda
- MAS `searchParams.get('empresaId')` da página atual continua igual
- Página não fetcheia dados novos da nova empresa
- Resultado: stale data até reload manual

**Correção proposta — modificar `setCurrentEmpresa` no contexto:**

```typescript
const setCurrentEmpresa = useCallback((id: string) => {
  setCurrentEmpresaIdState(id)
  // state + localStorage + cookie (já existe)
  ...
  // NOVO: sincronizar URL se contém ?empresaId= ou /empresas/[oldId]/...
  const oldEmpresaPathMatch = pathname.match(PATH_EMPRESA_RE)
  if (oldEmpresaPathMatch) {
    // Path tem /empresas/[id]/* — navega pra /empresas/[newId]/* mantendo resto
    const newPath = pathname.replace(oldEmpresaPathMatch[1], id)
    router.push(`${newPath}${qs ? `?${qs}` : ''}`)
  } else {
    // Path global — sincroniza ?empresaId= e força refresh
    const params = new URLSearchParams(searchParams.toString())
    params.set('empresaId', id)
    router.replace(`${pathname}?${params.toString()}`)
    router.refresh() // re-render server components
  }
}, [pathname, searchParams, router])
```

Atenção: o context já tem `usePathname` e `useSearchParams`. Só falta
adicionar `useRouter`.
