### Achados críticos

1. **Sparkline 12 meses NÃO É VIÁVEL** — só 4 meses de dados na Cacula Mix
   (Fev-Mai/2026). Decisão: hero usa "últimos N meses dinâmico" (`Math.min(12, mesesComDados)`),
   com fallback gracioso pra 3 meses mínimo. Quando N < 3, oculta sparkline.

2. **`lifecycle` é STRING, não enum** — valores reais: `EFFECTED` (OFX/pago manual no
   extrato), `PAYABLE` (conta a pagar — pode ter `paymentDate` quando paga, mas mantém
   lifecycle), `RECEIVABLE` (conta a receber).

3. **0 RECEIVABLE pagos em todos os meses** — Yussef ainda não usa contas a receber
   (entradas vêm via OFX como `lifecycle=EFFECTED` + `type=CREDIT`). Implicação pro
   Fluxo de Caixa:
   - **Realizado:** filtrar por `paymentDate ≠ NULL`, agrupar por `type` (CREDIT=entrada,
     DEBIT=saída), respeitando filtro `type ≠ TRANSFER` (Sprint 0.5).
   - **Previsto:** filtrar `lifecycle IN ('PAYABLE', 'RECEIVABLE')` + `paymentDate IS NULL` +
     `dueDate` no range. Por enquanto vai mostrar zero receivables — mas a lógica fica
     pronta pro dia que Yussef começar a usar A/R.

4. **profit sao borja com 285 pagos sem distribuição mensal calculada** — query
   adicional vai sair na Fase 3 (Fluxo de Caixa) mas indica boa fonte de dados pra
   testar relatórios fora da Cacula.

5. **profit itaqui zerado** — não vai mostrar dados em preview cards. Empty states
   precisam ser graciosos.

## 7. Riscos identificados

| Risco | Mitigação |
|---|---|
| Sparkline com poucos dados fica feio | Mostra "—" se < 3 meses; sparkline aparece só com ≥3 meses |
| profit itaqui sem dados pagos quebra cards | Empty state gracioso em cada card preview |
| Recharts SSR warning width(-1) | Pattern wrapper `'use client'` + `dynamic({ssr:false})` já validado no dashboard |
| Drill-down modal performance com muitos dados | Paginar (50 por vez) + skeleton no carregamento |
| Multi-tenant sneak: confundir empresaId | Reusar query builders centralizados (`lib/cashflow/query.ts` — sprint 0.5). Funções puras com `companyId` obrigatório |
| `lifecycle` enum dependency | Tratar como string union em TypeScript types, sem migration |
