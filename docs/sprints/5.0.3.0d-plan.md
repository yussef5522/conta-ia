### 2.2 Mobile drawer/sheet — instalar componente novo?

Spec pede "modais viram bottom-sheets em <768px". Hoje o projeto NÃO tem
componente `<Sheet>` nem `<Drawer>` (eu vi `components/ui/sheet.tsx` —
deixa eu confirmar abaixo).

Opções:
- **A (Recomendado)** Usar Radix `<Sheet>` se já existe; caso contrário,
  reaproveitar `<Dialog>` existente com classes responsive (`sm:max-w-none
  sm:rounded-none sm:h-screen sm:m-0`) — visual fica próximo de bottom-sheet.
- B Instalar `vaul` (drawer puro mobile-friendly, usado pelo shadcn) — +6kb.
- C Manter `<Dialog>` sem virar bottom-sheet — UX mobile pior.

**Vou recomendar A** se `sheet.tsx` já existe (90% dos casos). Confirmo
na Fase d3 e ajusto.

### 2.3 Playwright — quais browsers?

Spec sugere instalar chromium + webkit + firefox. Mas:
- Time de CI é 3× maior com 3 browsers
- Você usa Mac/iPhone (Safari) → webkit é o que mais importa
- Firefox tem usuário minoritário no Brasil

**Recomendação:** rodar local em chromium + webkit; CI em chromium-only
(rápido) com webkit em PRs marcados como release. Yussef escolhe.

### 2.4 Virtualization threshold

Spec diz "> 100 linhas". Mas Cacula tem 94 — vai funcionar pra empresas
maiores. Vou seedar fixture sintética de 1000 linhas pra teste local
SEM tocar prod.

### 2.5 Testes E2E — autenticação

Cenários E2E precisam de usuário logado. Opções:
- Reusar `admin@contaia.com.br / ContaIA@2025` do seed
- Criar `e2e@caixaos.com.br` dedicado (mais limpo, dados isolados)
- Mock JWT no Playwright

**Recomendação:** usuário de seed dedicado em `prisma/seeds/e2e-user.ts`.
Senha em env var (não commit).

### 2.6 GitHub Actions — qual ambiente?

Spec sugere postgres service em GH Actions + migrate + seed. Mas:
- E2E precisa de Anthropic API key (Claude Haiku no Excel detect)
- Build precisa de BrasilAPI (offline ok mas alguns features chamam)
- Tests rodando contra DB postgres efêmero exigem setup completo

**Recomendação:** CI minimal — só roda **unit tests** + **build** + **TS check**
em cada PR. E2E completo roda apenas em release/staging (manual `workflow_dispatch`
ou cron noturno). Razão: complexidade de orquestrar Anthropic + Postgres + browsers
em GH Actions free tier é alta. Yussef decide.
